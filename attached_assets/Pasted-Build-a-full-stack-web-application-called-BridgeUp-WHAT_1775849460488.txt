Build a full-stack web application called BridgeUp.

---

WHAT BRIDGEUP IS

BridgeUp is a cross-institutional platform connecting HSG (University of 
St. Gallen) business students with ETH Zurich technical students to form 
verified, recruiter-facing collaborative competition records. It is a 
professional, recruiter-grade platform — not a social network. Think of 
it as the verified project layer that LinkedIn and GitHub are missing for 
cross-disciplinary student work.

---

TECH STACK

- React with React Router for all routing and navigation
- Tailwind CSS for styling
- Firebase for authentication (email/password) and Firestore as the database
- All Firebase credentials must be stored as environment variables, 
  never hardcoded
- EmailJS to send the sponsor express interest form submission to 
  gfarruggio05@gmail.com — store the EmailJS service ID, template ID, 
  and public key as environment variables
- No external animation libraries — use pure CSS, Canvas API, and 
  IntersectionObserver only

---

DESIGN SYSTEM — CINVEN-INSPIRED

The visual language must match the authority and clarity of top-tier 
European private equity and finance firms like cinven.com. This is a 
light-mode site — clean white base with a strong red accent and 
near-black text.

Colors:
- Background: #FFFFFF (clean white base)
- Primary text: #1A1A1A (near-black charcoal)
- Accent / brand red: #E4002B (used on CTAs, hover states, active nav 
  links, and left border reveals on cards)
- Secondary accent: #C8001F (hover state on red elements)
- Muted text: #6B6B6B (captions, metadata, secondary labels)
- Card background: #F7F7F7 (subtle depth against white)
- Border: #E0E0E0 (dividers and card borders)
- Dark section background: #111111 (footer and high-contrast CTA 
  sections only, with white text)
- Nav background on scroll: rgba(255,255,255,0.92) with 
  backdrop-filter: blur(12px)

Typography:
- Display headings: Playfair Display (serif) — hero titles and section 
  headers
- Body and UI: Inter or DM Sans (clean sans-serif)
- Heading scale: H1 64px, H2 40px, H3 28px, body 16px, caption 13px
- Letter spacing on all-caps labels: 0.12em
- Line height on body: 1.7

Layout principles:
- Full-bleed sections with generous vertical padding (min 120px top 
  and bottom per section)
- Max content width: 1200px, centered, with 48px horizontal padding
- Thin horizontal rules (#E0E0E0) as section dividers — no heavy 
  gradient blocks
- Grid: 12-column, content typically spanning 8 columns centered or 
  split 6+6
- All text left-aligned in content sections, centered only in hero 
  and CTA sections

Animations (all must respect prefers-reduced-motion):
- Hero headline: word-by-word fade in and slide up, 80ms stagger 
  between words
- Scroll-triggered sections: fade in + slide up, 100ms stagger between 
  child elements, triggered by IntersectionObserver at 20% visibility
- Navbar: transparent on load, transitions to rgba(255,255,255,0.92) 
  with backdrop-filter: blur(12px) after 60px scroll
- Cards: on hover, lift with box-shadow and reveal a left border in 
  accent red (#E4002B)
- Page transitions: fade out on navigate, fade in on new page load
- Stats and numbers: count-up animation when scrolled into view
- Marquee ticker: horizontal scrolling text strip between hero and 
  the first content section
- Hero background: three layered effects — a perspective grid drawn 
  on Canvas using #E0E0E0 lines on white, slow-moving aurora orbs 
  in pale red and pale grey at very low opacity, and a noise texture 
  overlay. These three layers must be fixed-position and extend as a 
  subtle persistent background across the entire site at reduced 
  opacity.

Micro-interactions:
- Button press: slight scale down (0.97) on active state
- Input focus: accent red ring, no default browser outline
- All interactive elements have a 200ms ease transition
